PROJECT OVERVIEW
====================================================

The project solves the problem of centralized legal document verification systems that are vulnerable to tampering, unauthorized modification, lack of transparency, insider attacks, and dependency on centralized authorities.

The proposed system uses blockchain and decentralized storage to create a tamper-proof document verification platform.

Main functionalities include:

- Uploading legal documents

====================================================
SYSTEM WORKFLOW
====================================================

1. User uploads document through Streamlit frontend.
2. Flask backend receives document.
3. SHA-256 hash is generated.
4. File is uploaded to IPFS.
5. IPFS returns CID.
6. Smart contract stores:
   - filename
   - hash
   - timestamp
   - uploader address
   - version number
   - IPFS CID
7. Blockchain transaction is recorded.
8. User can later verify document integrity by uploading file again.
9. System recomputes hash and compares with blockchain records.
10. Verification result is displayed.
